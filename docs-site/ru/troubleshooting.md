# Решение проблем

## Нет транскрипции или пустой вывод

1. Убедитесь, что в настройках приложения указан API-ключ транскрипции.
2. Убедитесь, что выбранное устройство ввода звука действительно издает звук.
3. В Linux/PipeWire попробуйте установить рабочий микрофон как устройство ввода по умолчанию в системе и выберите `Default` в Voxis.
4. Если включен VAD, переключите бэкенд VAD на `Off (no filtering)` или настройте onset/hangover/prefill так, чтобы речь не обрезалась.
5. Помните, что записи короче минимальной длительности (по умолчанию 300 мс после VAD) отбрасываются до отправки запроса к API.
6. Проверьте журналы приложения (логи) в каталоге `logs/` внутри папки конфигурации платформы и файлы отладки в `debug/` при включенном режиме отладки.

## Неправильный микрофон в Linux

Имена устройств, отображаемые ALSA/CPAL, могут отличаться от имен, отображаемых KDE/PipeWire. Рабочий ввод PipeWire можно выбрать в качестве системного по умолчанию с помощью `wpctl set-default <id>`, тогда Voxis сможет использовать `Default`.

Полезные команды:

```bash
wpctl status
pactl list sources short
arecord -l
```

## Разрешения (Permissions)

Для записи требуется разрешение на использование микрофона. Автоматическая печать и глобальные горячие клавиши также могут потребовать разрешений ОС на доступность (accessibility) или ввод. Приложение включает команды для запроса разрешений и баннер, который может запрашивать доступ к микрофону/доступности и открывать соответствующие системные настройки, где это поддерживается.

## Изменения темы не отображаются

Пользовательские темы сохраняются при запуске. Если тема v2 уже существует в каталоге пользовательских тем, обновление только встроенной копии не перезапишет её. Скопируйте обновленные `theme.js` и `theme.json` в каталог пользовательских тем, затем перезагрузите/выберите тему снова или используйте Reload + Preview.

## Ошибка отсутствия API-ключа после установки переменных окружения

Приложение хранит API-ключи в локальной конфигурации и не считывает автоматически `GROQ_API_KEY` или `OPENAI_API_KEY` из оболочки (shell). Введите ключи в Настройках.

## Путаница с выбором провайдера

Настройка провайдера транскрипции предлагает ярлыки Groq и OpenAI, но текущий клиент транскрипции не маршрутизирует по этому ярлыку; он использует стандартный URL, совместимый с Groq, если не установлена `api_url_override` для тестов или кастомной сборки. Используйте ключ Groq `gsk_...` для стандартного endpoint. Ключ OpenAI `sk-...` является лишь примером формата учетных данных OpenAI. Настройки LLM-провайдеров отделены от транскрипции и поддерживают встроенных провайдеров Groq, OpenAI и OpenRouter, а также пользовательские определения.

## Ошибки сборки в Linux

Установите зависимости разработки Tauri/WebKitGTK и аудио для вашего дистрибутива. Workflow тестов GitHub перечисляет пакеты Ubuntu, используемые в CI: `libgtk-3-dev`, `libwebkit2gtk-4.1-dev`, `libappindicator3-dev`, `librsvg2-dev`, `patchelf` и `libasound2-dev`.
