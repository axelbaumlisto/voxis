# Установка

## Требования

- Bun для фронтенд-скриптов.
- Rust и Cargo для бэкенда Tauri.
- Системные зависимости, требуемые Tauri v2, WebKitGTK, appindicator, рендерингом SVG, `patchelf` и ALSA в Linux.
- API-ключ транскрипции, настроенный локально в настройках приложения.

В CI репозитория устанавливаются следующие пакеты Ubuntu для тестов бэкенда:

```bash
sudo apt-get install -y libgtk-3-dev libwebkit2gtk-4.1-dev libappindicator3-dev librsvg2-dev patchelf libasound2-dev
```

## Настройка для разработки

```bash
bun install
bun run tauri dev
```

Конфигурация разработчика Tauri запускает `bun run dev` и обслуживает фронтенд на `http://localhost:5173`.

## Сборка

```bash
bun run tauri build
```

Конфигурация сборки Tauri сначала запускает `bun run build`. Этот скрипт выполняет `bun run build:themes`, проверку типов TypeScript (`tsc`) и рабочую сборку Vite.

Настроенные цели сборки Tauri: `app`, `dmg`, `deb` и `rpm`. В Linux готовые пакеты сохраняются в `src-tauri/target/release/bundle/`.

## Заметки по пакетам Linux

Конфигурация сборки Tauri включает форматы Debian и RPM. Результат сборки зависит от хоста и установленных инструментов; установите зависимости для Linux, указанные выше, перед сборкой или запуском тестов на системах на базе Ubuntu.

## API-ключи

По умолчанию Voxis использует облачную транскрипцию. Для стандартного endpoint Groq:

1. Создайте аккаунт и войдите на [console.groq.com](https://console.groq.com/).
2. Откройте **API Keys** в консоли Groq.
3. Создайте новый ключ и скопируйте его (показан только один раз).
4. Вставьте его в **Settings → Provider → API Key**.
5. Используйте стандартную модель `whisper-large-v3` или выберите `whisper-large-v3-turbo` в Настройках.

Формат ключа:

- Вставляйте только сам ключ — без префикса `Bearer `, без кавычек и лишних пробелов.
- Ключи Groq обычно выглядят как `gsk_...`.
- Ключи OpenAI обычно выглядят как `sk-...`; это лишь пример формата ключей OpenAI. В настройках присутствует выбор OpenAI, но текущий клиент транскрипции использует конечную точку, совместимую с Groq, если не установлена переменная `api_url_override` для тестов или кастомной сборки.

Проверьте текущие лимиты и цены Groq в консоли.

Не добавляйте API-ключи в коммиты. Настраивайте ключи для транскрипции и (опционально) для LLM через интерфейс приложения; код сохраняет их в локальном хранилище конфигурации, а не загружает автоматически из переменных окружения. См. [Безопасность](security.md).
