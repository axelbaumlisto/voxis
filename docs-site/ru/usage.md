# Использование

## Процесс записи

1. Нажмите настроенную горячую клавишу (hotkey). По умолчанию это правый Ctrl (`ctrl_r`) с задержкой удержания 300 мс.
2. Voxis начнет запись с выбранного устройства ввода звука (по умолчанию `Default`, если не изменено).
3. В режиме удержания отпустите горячую клавишу для остановки. В режиме переключения (toggle) нажмите один раз для начала и еще раз для остановки.
4. Захваченный звук фильтруется через VAD (если выбран бэкенд VAD), затем ставится в очередь на транскрипцию.
5. Записи короче минимальной длительности (по умолчанию 300 мс после VAD) отбрасываются до отправки запроса к API.
6. Приложение применяет словарные замены, фильтрацию галлюцинаций для известных фраз без речи, обрабатывает запрос на перевод на английский (опционально), применяет постобработку LLM (опционально) и форматирует вывод.
7. Итоговый текст автоматически печатается (auto-typed), если эта функция включена. Иначе приложение использует буфер обмена с помощью настроенных сочетаний клавиш вставки и пытается восстановить предыдущее содержимое буфера обмена.

Оверлей отображает состояния бездействия, записи, транскрипции и ошибки. Он также поддерживает диктовку нажатием/отпусканием на непрозрачных пикселях канваса, поэтому прозрачные области более крупных тем-аквариумов не запускают запись.

## Страницы и ярлыки (шорткаты)

Основные страницы приложения: History (`/` и `/history`), Dictionary (`/dictionary`) и Settings (`/settings`). Поток первого запуска использует `/onboarding`.

Шорткаты приложения: `h` для Истории, `w` для Словаря, `s` для Настроек, `c` для копирования последней транскрипции и `Esc` для скрытия окна. Обработчик шорткатов игнорирует нажатия клавиш при вводе в текстовые поля.

## История, словарь и неудачные записи

История транскрипций включена по умолчанию (`history_enabled` по умолчанию true в локальной конфигурации) и сохраняет транскрипции в локальном хранилище. Неудачные транскрипции хранятся в папке `failed_audio/` в виде WAV-аудио и JSON-метаданных; хранилище сохраняет до трех записей, и на странице Истории их можно повторить или удалить.

![History page with mock transcription entries](../images/history.png)

Словарные записи заменяют распознанный текст на предпочтительное написание или термины. Предложения к изучению можно отключить, оставить ожидающими подтверждения вручную или применять автоматически, в зависимости от настроек словаря.

## Расположение данных во время выполнения

Данные сохраняются в каталоге конфигурации платформы в папке `voxis`, например `~/.config/voxis/` в Linux. Важные файлы и каталоги включают `config.db`, `history.db`, `dictionary.txt`, `corrections.db`, `providers.db`, `prompts.db`, `failed_audio/`, `debug/`, `logs/` и `themes/`.
